# Governance Glossary

Short definitions for terms used repeatedly in this repo.

## Active admin

An admin who still has the relevant role, can be reached through the recorded channels, and can acknowledge governance matters within the continuity-policy window.

Canonical source: `docs/governance/continuity-and-succession.md`

## Approval

An explicit public admin signoff in the relevant issue or PR thread.

## Bootstrap phase

The current early project stage where governance is still admin-led and deliberately lightweight.

## Canonical

The document or record treated as the authoritative source for a topic.

## Continuity issue

A public issue opened when admin availability, access coverage, or recovery capacity is at risk.

Canonical source: `docs/governance/continuity-and-succession.md`

## Decision log

The durable record of accepted governance and financial-control decisions.

Canonical source: `docs/decision-log.md`

## Emergency action

A narrow temporary action taken before the normal process finishes because delay would create material risk.

Emergency action should still be followed by public recordkeeping and retrospective review.

## Public ledger

The canonical public record of project money movement.

During the bootstrap phase, this can live in the repo under `docs/transparency/ledger/` until an external provider is adopted.

## Quorum

The minimum number of eligible admins who must participate for a formal decision to count.

Current bootstrap quorum is 2 active admins.

Canonical source: `docs/governance/roles-and-voting.md`

## Ratification

The step where admins formally approve policy text and the accepted outcome gets recorded in the decision log.

## Recusal

Stepping out of review, voting, or approval because a conflict of interest makes participation inappropriate.

Canonical source: `docs/governance/conflict-of-interest.md`

## Reporting cycle issue

The monthly GitHub issue used to prepare and publish a transparency report for a specific month.

## Transparency report

The monthly public report summarizing governance decisions, financial activity, policy changes, and current risks.
