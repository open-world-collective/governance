# Governance Roadmap (Bootstrap v1)

This roadmap is the canonical high-level plan for governance work in this repository.

It is a planning document, not a binding promise, and it should be updated as the project learns what is actually sustainable.

## Format choice

Bootstrap roadmap format: milestone table with:

- a plain-English goal
- a current status
- a target review date
- a role-based owner
- measurable exit criteria

This format is intentionally simple. It is meant to show what the project is trying to complete next without pretending the roadmap is more stable than it is.

## Review cadence

- During the bootstrap phase, review this roadmap monthly.
- Open a public governance roadmap review issue for each monthly review cycle.
- If no milestone status changes, record `no roadmap change` rather than skipping the review.
- If the project outgrows monthly review, change the cadence through the normal governance process.
