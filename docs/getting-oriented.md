# Getting Oriented

This is the short version of how governance works here.

## What this repo is for

This repo is where Open World Collective writes down the rules that are supposed to outlast mood, memory, and personal control.

It is the canonical home for:

- why the project exists
- how governance decisions get made
- how financial approvals and transparency work
- how moderation and appeals are supposed to work
- how accepted decisions get recorded

It is not where technical implementation details live. Those belong in the technical repos that actually implement them.

## How a governance change usually happens

1. Someone opens an issue.
2. The policy text gets drafted or updated in the repo.
3. Admins review and approve it in public.
4. The accepted decision is recorded in `docs/decision-log.md`.

If text is merged but not ratified yet, treat it as merged draft text, not as a fully recorded governance decision.

## What matters most right now

This project is still in a bootstrap admin-led phase.

Current baseline:

- admins make formal governance decisions
- quorum is 2 active admins
- financial actions need 2-admin signoff and public record
- conflicts require disclosure and recusal
- continuity depends on keeping at least 2 active admins and a usable private runbook
- transparency reports are published monthly
