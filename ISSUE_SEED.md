# Governance Issue Seed

Use format `- Title :: Description`.

- Ratify initial charter scope :: Define and approve what governance repo owns in phase 1.
- Define admin roles and removal process :: Document admin powers, limits, and removal criteria.
- Define decision voting rules :: Specify majority vote mechanics, quorum expectations, and tie handling.
- Define financial approval policy :: Require two-admin signoff and documentation path for fund use.
- Open Collective transparency policy :: Define ledger usage, required metadata, and publication rules.
- Transparency report cadence :: Decide report frequency and assign publication ownership.
- Moderation and appeals policy :: Draft process for moderation actions and appeal timelines.
- Conflict-of-interest policy :: Define disclosure and recusal requirements for decisions.
- Continuity and succession policy :: Define account recovery and continuity if an admin becomes unavailable.
- Governance roadmap milestones :: Track bootstrap-to-expanded governance transition milestones.
